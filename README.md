# Azure Cloud Infrastructure & Network Security Lab

## Status
Ongoing

---

## Overview
Designed and implemented a cloud-based lab environment in Microsoft Azure to develop hands-on experience with virtual networking, secure infrastructure, and access control.

This lab focuses on building foundational cloud architecture while reinforcing networking and security principles, including virtual networks, subnets, and network security groups.

---

## Environment

- Microsoft Azure
- Virtual Networks (VNet)
- Subnets
- Network Security Groups (NSGs)
- Virtual Machines (Linux / Windows)

---

## Architecture

- Virtual Network with segmented subnets
- NSG applied at subnet and/or NIC level
- VM deployment within defined network boundaries
- Region-aligned resource configuration

---

## Core Concepts Applied

- Cloud networking and segmentation
- Access control using NSGs
- Resource dependency and regional alignment
- Infrastructure deployment workflows
- Secure-by-design principles

---

## Technical Implementation

- Created virtual networks and subnets within Azure
- Configured NSG inbound and outbound rules for traffic control
- Deployed virtual machines within the defined network
- Troubleshot connectivity issues related to region mismatch and network configuration

---

## Key Insights

- Region alignment is critical for resource visibility and deployment
- NSG configuration directly impacts connectivity and access control
- Cloud infrastructure requires structured planning before deployment
- Troubleshooting is essential to understanding cloud networking behavior
